## When to Use This Skill

Trigger this skill when the user asks to:
- "Write an ad for [Product]"
- "Audit this ad"
- "Give me some hooks for [Product]"
- "How should I test creatives for [Product]?"
- "Act as my creative strategist"

## Execution Workflow

### Scenario A: User wants to create a new ad
1. **Intake:** Do not write the ad yet. Ask the 5 Core Questions from `PRODUCT_INTAKE.md` or extract them from a provided URL.
2. **Persona Mapping:** Identify which of the Core 4 Archetypes (`PERSONA_SYSTEM.md`) this product targets.
3. **Angle Matrix:** Generate 3-5 distinct psychological angles using `ANGLE_GENERATION.md`. Present these to the user.
4. **Scripting:** Once the user selects an angle, write the full script using the 5-Beat Structure (`VIDEO_FRAMEWORK.md`). Include exact dialogue, visual direction, and text overlays for every beat.

### Scenario B: User wants to audit an existing ad
1. **Analyze:** Break the ad down into the 5 Beats (Hook, Villain Pivot, Mechanism, Proof, CTA).
2. **Score:** Evaluate the ad based on the North Star principle: "Would this make someone stop scrolling and buy?"
3. **Diagnose:** Identify the specific point of failure (e.g., "The mechanism is generic," "The proof is verbal, not visual").
4. **Prescribe:** Give exact, surgical instructions on how to fix it.

### Scenario C: User wants hooks
1. Ask for the target persona and the winning angle (if known).
2. Generate 10 distinct hooks using the proven formulas (Confession + Conspiracy, The Math Hook, Paradox, etc.).
3. Ensure every hook pays the "Skepticism Tax" and uses specific numbers.

## Key Rules (Always Apply)

1. **The Constructed Authority:** Personas must have specific backstories (e.g., "23 years," "3,000 procedures").
2. **The Price Anchor:** Always anchor against the most expensive, painful alternative (e.g., "$38,000 surgery vs $80 device").
3. **Specificity:** Use odd numbers (87%) and specific dollar amounts ($863).
4. **The Wind Test / Live Wipe:** Proof must be a physical, undeniable demonstration, not just a claim.
5. **Text Overlays:** Must appear within 2 seconds (silent viewers).
6. **Pacing:** Must be Fast or Medium-Fast. No dead air.
